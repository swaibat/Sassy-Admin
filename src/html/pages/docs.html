Get started

To view your Bootstrap Theme documentation, located in the docsdirectory and open docs.html in your favorite browser.all tutorials will be available plus the video on how to get started with sassy admin theme

Incase you want to access the default built files for thids project simply open the dist folder located at the root of the theme and you will be able to access all the built content HTML, CSS and JAVASCRIPT files

gulp workflow

We have used Gulp workflow for easy customization and automation of this project thats why we have included a Gulpfile and used Babeljs to support es6 in our project thats why the gulpfile is having an extention of babel gulpfile.babel.js

Before installation of the application make sure you have nodejs on your machine you can download if from official website nodejs.org, This will enable us install run and install necessary packages that have been used in our project

The run this command to install Gulp on your local machine

After make sure all theme dependencies are installed by runing the following command or else the themwe may fail to work

After you can modify and customize the app according to your requirements and desires and after run the project by simply typing gulp in terminal

if you have any question about the gulp workflow run $ gulp --help in commandline or visit gulpjs.com

Theme source code

The dist/ folder contains ready to view compiled theme

Simply open the source folder /src you will find HTML , JS , IMAGES and SCSS folders

NOTE:This folder is mearnt to be for customizing your theme which is later compiled to the dist folder not for viewing the theme
  • HTML This folder contains all our basic pages and partials that are included before compling and cutomizing of HTMl Is done here
  • IMAGES The folder includes image files to be used in our project like logo favicon and others
  • JS The folder contains all javascript files and libraries that have been used in our project
  • SCSS The folder contains scss files that are later compiled to css

The docs/ folder includes the source code for our documentation.

The remaining files not specifically mentioned above provide support for packages, license information, and development.
